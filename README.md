# 1 課題名
フックマークDB（≠ブックマーク）

# 2 課題内容（どんな内容か）
ブックマークDBではなく、フックマークDBを作成。画面越しに右フックすると速度を測り、その結果をDBに保存してくれる。

# 3 アプリのデプロイURL
https://renren.sakura.ne.jp/hook_mark/index.php

# 4 アプリのログイン用IDまたはPassword（ある場合）
なし

# 5 工夫した点・こだわった点
リアルタイム画像認識の活用：ブラウザのカメラからリアルタイムで、右フックを判定する画像認識ロジックを組み込んだ。
速度計算のアルゴリズム：単に認識するだけでなく、右拳の座標移動距離と時間から拳速を算出する計算式を作成。

# 6 難しかった点・次回トライしたいこと（又は機能）
ローカルと本番環境の環境差による予期せぬエラーに何度も直面し、原因特定のためのデバッグ作業に多大な時間を費やしました。。。
が、その分サーバー構成への理解が深まりました。
